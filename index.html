JavaScript определяет 7 типов данных:

  1. null
  2. undefined
  3. boolean
  4. string
  5. number
  6. object
  7. symbol (добавлен в ES6)

Все типы данных кроме объектов (objects) — примитивные.

Symbol как скрытый идентификатор

Символы создаются вызовом функции Symbol(), в которую можно передать описание (имя) символа. Он уникален.

             let id = Symbol("id") 
             let user = { 
                 name: "Вася", 
                 age: 30, 
                 [id]: 'unique hidden identifier'   
             };

Symbol как скрытый идентификатор

Symbol при итерации через Object.keys(), for будет скрыт. А вот Object.assign, в отличие от цикла for..in, копирует и строковые, и символьные свойства:

Технически символы скрыты не на 100%. С помощью Object.getOwnPropertySymbols(obj) можно получить все свойства объекта с ключами-символами. Также существует метод Reflect.ownKeys(obj), который возвращает все ключи объекта, включая символьные.

Оператор typeof

С помощью оператора typeof можно получить тип переменной: 

            let name = "Tom"
            console.log(typeof name); // string
        
            let income = 45.8
            console.log(typeof income) // number

Примитивные типы

Все типы данных в JavaScript, кроме объектов, являются неизменяемыми или иммутабельными (англ. immutable). Это значит, что их значения не могут быть модифицированы, а только перезаписаны новым другим значением. Например, строки нельзя корректировать посимвольно – их можно переписать только полностью. Значения таких типов называются "примитивными".

Массивы также имеют тип данных object

Фактически массивы и есть объекты с той лишь разницей, что свойства массива являются нумерованными индексами. 

            typeof [1,2,3] === “object”; // true

В JavaScript всё ведет себя, как объект, лишь за двумя исключениями — null и undefined.

            false.toString(); // 'false'
            [1, 2, 3].toString(); // '1,2,3'
            function Foo(){}
            Foo.bar = 1;
             Foo.bar; // 1

Number тоже можно использовать в качестве обьектов

            2.toString(); // вызывает SyntaxError

Есть несколько способов обойти этот недостаток и любой из них можно использовать для того, чтобы работать с числами, как с объектами: 

            2..toString(); // вторая точка распознаётся корректно
            2 .toString(); // обратите внимание на пробел перед точкой
            (2).toString(); // двойка вычисляется заранее

Значения Number

Числовые литералы number могут быть выражены в других формах, таких как, двоичная, восьмеричная, и шестнадцатеричная.

Кроме того, JavaScript содержит специальные числовые значения:

  • NaN -- не число или ошибка вычислений
  • Infinity -- положительная бесконечность (1 / 0)
  • -Infinity -- отрицательная бесконечность (-1 / 0)
  • ±0 -- положительный и отрицательный 0 (0 / -1 || 0 * -1)

0.1 + 0.2 === 0.3 // false

Маленькие дробные числа

Математически, что результатом выражения должно быть true. Почему же в результате получается false?

Если по-простому, представления чисел 0.1 и 0.2 в бинарном виде с плавающей точкой не совсем точные, поэтому когда мы их складываем, результат не совсем 0.3. Это действительно близко: 0.30000000000000004, но если сравнение не прошло, "близко" уже не имеет значения.

Как быть, спросишь ты? Или не спросишь. Но я отвечу

Number.EPSILON

Самая общепринятая практика использование миниатюрной "ошибки округления" как допуска для сравнения.
Это малюсенькое значение часто называют "машинной эпсилон," которое составляет 2^-52 (2.220446049250313e-16) для числового типа number в JavaScript.

Number.EPSILON 

                function numbersCloseEnoughToEqual(n1,n2) {
                    return Math.abs( n1 - n2 ) < Number.EPSILON;
                }
                let a = 0.1 + 0.2
                let b = 0.3
                numbersCloseEnoughToEqual( a, b ) // true

У переменных нет типа — тип есть у значений

null VS undefined

Переменная без присвоения ей начального значения - undefined 

            var isAlive;
            console.log(isAlive); // выведет undefined

null VS undefined

Переменная имеет некоторое неопределенное значение (не число, не строка, не логическое значение), но все-таки имеет значение- null 

            isAlive = null;
            console.log(isAlive); // null

Тип данных Object

Объект состоит из многих значений (либо элементарных, таких как числа и строки, либо сложных, например других объектов), которые называются свойствами объекта. Доступ к свойствам осуществляется по имени свойства (иногда говорят «по ключу»).

Доступ к свойствам

Получить доступ к свойствам объекта можно двумя способами: используя либо точечную нотацию, либо запись квадратными скобками. 

            let someNewObject = {name: 'kitten'}
            foo.name; // kitten
            foo['name']; // kitten 
            var get = 'name';
            foo[get]; // kitten

Строки VS массивы

Строки имеют поверхностное сходство с массивами для них работают свойство .length, методы .indexOf() .concat().

Строки в JavaScript неизменяемы, тогда как массивы могут меняться. Для строки нет методов, которые меняют ее содержимое на месте, метод лучше создаст новую строку и вернет ее нам.

Клоны переменных

Простые величины null, undefined, string, number, boolean, и ES6’s symbol всегда копируются по значению. 

             let a = 2 
             let b = a 
             b++ 
             console.log(a, b)
                // 2, 3

Клоны переменных

Сложные величины объекты (в том числе массивы) и функции всегда копируются по ссылке.

             let c = [1,2,3] 
             let d = c 
             d.push( 4 ) 
             console.log(c, d)
                // [1,2,3,4]
